# Development Guide

## 開発フロー

- ユーザに依頼をされたら、分析を行ったうえで TODO.md に実装計画を記述する
- コード変更を行ったら **必ずテストを追加/変更する** こと
- カバレッジを常に90%以上をキープすること
- `deno task test` ですべてのテストを通すこと
- `deno lint` で検出されるエラーを0件にすること
- コミットは論理的単位で適切に行うこと

## TODO管理方針

- 完了したタスクはTODO.mdから削除する。過去のTODOを知りたい場合は**git履歴**から確認できる
- 過去のタスクを参照する必要がある場合は `git log -p TODO.md` を使用

### 過去のタスク参照方法

```bash
# TODO.mdの履歴を見る
git log -p TODO.md

# 特定のコミット時点のTODO.mdを見る
git show <commit-hash>:TODO.md

# 削除前の最後のアーカイブを見る（2026-01-12時点）
git show c37630c:TODO_ARCHIVE.md
```

### TODO.mdの更新ルール

- タスク完了時: TODO.mdから該当タスクを削除するか、チェックボックスをチェック
- 新規タスク追加時: 優先度と実装計画を明記
- 大きなマイルストーン完了時: git commitでタスク完了を記録
- 完了済み、あるいは放置されたTODOがある場合削除すること
