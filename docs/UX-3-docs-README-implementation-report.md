## 主な改善点

### 1. ドキュメントの発見性向上

**Before**:

- docs/ にアクセスしてもファイル一覧しか見えない
- どれを読むべきか分からない
- ユーザー向けドキュメントが存在しない

**After**:

- ユーザー向け vs 開発者向けが明確に分離
- 「次に何を読むべきか」が段階的に示される
- 既存ドキュメント（README.md、uploader.example.yaml）への誘導

### 2. 段階的な学習パス

ユーザーのレベル別に最適な学習順序を提示:

1. **初めての方**: README → 設定ファイル作成 → 実行 → FAQ
2. **詳しく知りたい方**: example.yaml → Configuration セクション → Use Cases
3. **開発者**: CLAUDE.md → SPEC.md → implementation/ → UX_REVIEW_REPORT.md

### 3. トラブルシューティングのナビゲーション

「困ったときは」テーブルで問題から解決策へ素早くアクセス:

| 問題                 | 参照先                        |
| -------------------- | ----------------------------- |
| 動かない             | README.md - Requirements      |
| 設定方法が分からない | README.md - Configuration     |
| プロトコルの選び方   | README.md - プロトコル        |
| Windowsで動かない    | README.md - Windowsの注意事項 |
| 認証エラー           | README.md - 認証方式          |
